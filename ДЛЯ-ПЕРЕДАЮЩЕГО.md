# Инструкция для вас (кто передаёт сайт)

Краткий чеклист, что сделать, чтобы человек стал **реальным владельцем**, а не просто получил ссылку.

## Перед отправкой

1. Узнайте у партнёра:
   - логин **GitHub**;
   - email **Vercel** (тот же, что при регистрации на vercel.com).

2. Отправьте ему:
   - архив **`debit-showcase-handoff.zip`** (вся папка с сайтом);
   - файл **`ПЕРЕДАЧА-ВЛАДЕЛЬЦУ.md`** (главная инструкция);
   - ссылку на сайт: https://debit-cards-ru.vercel.app

## GitHub — передать код

1. https://github.com/hlebbbik777-bot/dc  
2. **Settings** → **Danger Zone** → **Transfer ownership**  
3. Ввести логин партнёра → он подтверждает в почте.

*Если Transfer недоступен:* **Collaborators** → Add → **Admin**, потом партнёр делает fork и вы удаляете старый репо (хуже, чем Transfer).

## Vercel — передать хостинг

1. https://vercel.com → проект **debit-cards-ru**  
2. **Settings** → **Transfer Project** → email/команда партнёра.

*Если Transfer нет:* партнёр импортирует **свой** GitHub-репозиторий после Transfer на GitHub (см. ПЕРЕДАЧА-ВЛАДЕЛЬЦУ.md, шаг C).

## Партнёрские деньги

Обязательно предупредите: в `offers.js` стоят **ваши** ссылки getlink (`aff_id=221509`).  
Пока партнёр их не заменит — конверсии ваши.

## После передачи (рекомендуется)

- Удалить у себя проект `debit-cards-ru` на Vercel (когда у партнёра свой деплой работает).  
- Или отозвать себе Admin в GitHub, если оставили Collaborator.
